# Nuxt 生态系统的文档撰写者

撰写博客文章和文档的指导原则，遵循官方 Nuxt 网站中的模式。

## 使用场景

- 为 Nuxt 生态系统项目撰写博客文章
- 创建或编辑文档页面
- 确保内容中写作风格的一致性

## 写作标准

**覆盖规则**：撰写文档时，需保持语法正确和句子完整。此处不适用“为简洁牺牲语法”的规则。

文档必须：

- 语法正确
- 表达清晰且无歧义
- 标点符号使用得当
- 使用完整句子（而非片段）

简洁性仍然重要，但绝不以牺牲清晰度或准确性为代价。

## 相关技能

如需了解组件与语法详情，请使用以下技能：

| 技能             | 用途                                             |
| ---------------- | ------------------------------------------------ |
| **nuxt-content** | MDC 语法、叙述性组件、代码高亮                   |
| **nuxt-ui**      | 组件属性、主题设置、UI 模式                      |

## 快速参考

### 写作模式

| 模式         | 示例                                               |
| ------------ | -------------------------------------------------- |
| 主语优先     | “`useFetch` 可组合函数处理数据获取。”              |
| 命令式       | “将以下内容添加到 `nuxt.config.ts`。”              |
| 上下文相关   | “在使用认证时，应配置……”                           |

### 情态动词

| 动词     | 含义         |
| -------- | ------------ |
| `can`    | 可选         |
| `should` | 建议         |
| `must`   | 必须         |

### 组件模式（何时使用）

| 需求              | 组件                              |
| ----------------- | --------------------------------- |
| 信息旁注          | `::note`                          |
| 建议              | `::tip`                           |
| 警告              | `::warning`                       |
| 必备项            | `::important`                     |
| 行动号召          | `:u-button{to="..." label="..."}` |
| 多源代码块        | `::code-group`                    |

> 组件属性详情请参阅 **nuxt-ui** 技能

## 标题

- **H1 (`#`)**：不使用反引号——它们无法正常渲染
- **H2 至 H4**：反引号可以正常使用

- [ ] 先解释再写代码

- [ ] 使用恰当的提示类型
- [ ] H1 标题中不使用反引号
